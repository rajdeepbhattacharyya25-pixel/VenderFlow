# Feature Building Workflow

1. **Analyze existing project**: Understand the current state and requirements.
2. **Create change plan**: detailed breakdown of required changes (Database, Backend, Frontend).
3. **Present plan to user**: Explain the changes and dependencies.
4. **Get approval**: Wait for user confirmation.
5. **Apply changes**: Implement the code changes.
6. **Test**: Verify the changes work as expected.
7. **Show preview**: Demonstrate the new feature to the user.
